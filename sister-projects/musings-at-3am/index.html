This website is simply a test website for me to practice and showcase my newly learnt skill of HTML5 in tandem with CSS3. Therefore, the emphasis of this website is not on the content (e.g., the videos, the text, etc.), but on the coding and the work put into it to make both the source code and the compiled result look picture perfect and nice.

I am not a huge fan of web programming, let alone PHP and Javascript, for those things are like web-bloat. Therefore, I promise you that everything you see in here is done with nothing else but HTML5 and CSS3 sorcery; there's not even a single line of Javascript written, not even for the cover video! Best of all, you have the right to view said HTML5 and CSS3 sorcery and inspect it all you want right here.

CSS rules in the stylesheet are organized by using Fortran-style labels. It's hard to explain what I mean here, but try looking at the stylesheet and you'll see how I structured it.

This website can actually morph into two forms: desktop and mobile. That's right — it's like we have a desktop site and a mobile site, where the desktop site contains a lot more aesthetics and glitter fluff than the mobile site. However, in this website, the desktop and mobile sites are combined into one; the content stays the same, hardly any is added or removed, but the CSS is starkly different. This is thanks to what we call @media queries.

Now, here's the ridiculous bit, the only way the website is going to know whether you are on a desktop or mobile setup is by looking at the ratio of your viewport's width to height (orientation) and your viewport's width and run the necessary logic to select the CSS rules to compile. Therefore, if you are on a desktop and you resize your viewport to the size of a quintessential tablet device, you will get a mobile site, and vice versa for mobile devices.

It is an axiom that I'm still pretty new at HTML5 and CSS3. So, for all the coding style you are to encounter here, I daresay it can always be improved, and that is something I am bound to do. I also have to admit that the source code and semantic structure of this website must be refactored and improved by a lot; this is even though the resulting compiled output looks eye-pleasing enough. This is something I must work on in the future. An overview of the things to be improved can be found in the TODO list commented at the end of this HTML file.

I give credit to every aspect of the website (e.g., cover video, fonts) within the source code and the footing. So, you can check out their names and how they came to be right there.

So, what are you waiting for? Take a look at the source code and inspect the elements (F12 on most browsers)! üòù
